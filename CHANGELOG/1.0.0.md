# 1.0.0 - 24/06/2026

After over a year of active development within UKCEH's FDRI programme, time-stream 1.0.0 marks the first stable,
versioned public release. The library provides a high-level API for working with environmental time series data
built on [Polars](https://pola.rs/), with a focus maintaining the integrity of the temporal properties time series data.

## Features

### Core

- **`TimeFrame`** - the central data structure wrapping a Polars DataFrame  with an associated periodicity and
time anchor, enforcing a consistent schema for time series operations.
- **`Period`** - a rich period/frequency abstraction supporting both fixed-duration (sub-daily through multi-year)
and calendar-aligned (monthly, annual) periods, with ISO 8601 duration support.

### Flagging

- Bitwise and categorical flag systems for recording QC and processing decisions alongside the data.
- Flag columns can be attached to any `TimeFrame` column.
- Support for encoding and decoding flag values, and auto-generation of bitwise flag integers from a list of named
categories.
- Filter a `TimeFrame` by flag values to subset data based on QC or processing state.

### Aggregation

- Standard temporal aggregation via `TimeFrame.aggregate()`, with support for custom time windows, time anchoring,
and configurable missing-data criteria.
- Rolling aggregation via `TimeFrame.rolling_aggregate()` with trailing, leading, and centre alignment.
- Built-in aggregation functions, including: mean, sum, min, max, standard deviation, angular mean, and count.

### Quality Control

- Extensible QC framework (`QCCheck`) with a registration system allowing custom checks to be defined and reused by
name.
- Built-in checks, including: range, flatline, and spike detection.
- QC results returned as a Polars `Series` or written directly into a flag column on the `TimeFrame`.

### Infilling

- Infill framework (`InfillMethod`) for filling gaps in time series, with a registration system for custom methods.
- Built-in interpolation methods, including: linear, quadratic, cubic, B-spline, Akima, and PCHIP.
- Alternative-data infill from a secondary column or DataFrame, with an optional correction factor.
- Configurable maximum gap size and observation interval constraints to limit where infilling is applied.
